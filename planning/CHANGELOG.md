## [Unreleased]

### Added

- **Restrição de processamento — GDPR Art. 18 (`lib/core/services/processing_restriction_service.dart`, `settings_screen.dart`, `app_user.dart`):** implementado o direito de restrição de processamento conforme GDPR Art. 18. Novo `accountStatus: 'restricted'` que congela o processamento de dados sem os apagar — `canSendContent` já retorna `false` para qualquer status diferente de `active`, portanto cartas, cápsulas, comentários e likes ficam automaticamente bloqueados. Novo `ProcessingRestrictionService` com métodos `restrictProcessing()` (define `accountStatus: 'restricted'` + `restrictionRequestedAt`) e `liftRestriction()` (volta a `active`, remove timestamp). UI no Settings com menu item dinâmico (ícone, label e subtitle mudam conforme estado) + diálogo de confirmação com feedback SnackBar. Getter `isRestricted` adicionado ao `AppUser`. 8 chaves i18n novas em 4 idiomas (en/pt/pt-BR/es).

- **Access logs — Marco Civil Art. 15 (`functions/src/log_access.ts`, `purge_old_access_logs.ts`, `lib/core/services/access_log_service.dart`, `auth_repository.dart`):** nova Cloud Function callable `logAccess` que regista cada autenticação na collection `accessLogs`. IP capturado server-side via `x-forwarded-for` / `rawRequest.ip` — dados que o cliente não consegue obter. UID pseudonimizado com `hashUid()` (HMAC-SHA-256). Campos: `uidHash`, `ip`, `platform` (ios/android/web), `authMethod` (email/google/apple), `createdAt`. Nova Cloud Function scheduled `purgeOldAccessLogs` (diariamente 05:30 UTC) que deleta logs com `createdAt` > 180 dias (6 meses), cumprindo o compromisso da Seção 9 da Política de Privacidade. Novo serviço client `AccessLogService.logLogin()` — fire-and-forget via `SafeCallable`, chamado em `auth_repository.dart` após cada login (email, Apple, Google). Firestore rules: `accessLogs` com `read, write: if false` (só Admin SDK). Constante `FirestoreCollections.accessLogs` adicionada.

- **Hash criptográfico em audit logs (`delete_account.ts`, `request_deletion.ts`, `export_user_data.ts`, `scheduled_deletion.ts`):** substituída `simpleHash()` (djb2, 32-bit, sem salt, colisões triviais) por `hashUid()` — HMAC-SHA-256 com salt estático (`whenote-audit-log-v1-2026`), output truncado a 16 hex chars (64 bits). Resistente a colisões e não-reversível. Aplicado em todos os 4 ficheiros de backend que pseudonimizam UIDs em `deletionAuditLogs` e `privacyRequestLogs`. `simpleHash()` marcada como `@deprecated` mas mantida temporariamente para referência. `scheduled_deletion.ts` corrigido: usava `uid.substring(0, 8) + "..."` (PII parcial em texto claro), agora usa `hashUid(uid)`.

- **Audit trail de privacidade completo (`privacy_log_service.dart`, `firestore.rules`, `settings_screen.dart`):** expandido o `PrivacyLogService` com 3 novos métodos: `logDeletionRequest()` (regista pedido de exclusão com modo e provider), `logDeletionCancellation()` (regista cancelamento de exclusão pendente), `logReauthentication()` (regista tentativa de re-auth social — Apple/Google). Todos os métodos seguem o padrão fire-and-forget com try/catch (best-effort). Chamadas injetadas em `settings_screen.dart`: `_executeAccountDeletion()` (success + failure), `_cancelDeletion()` (success + failure), `_handleSocialReauth()` (success + failure), `_handleConfirm()` / password re-auth (success + failure). Firestore rules atualizadas: whitelist de `type` expandida de `['export']` para `['export', 'complete_export', 'deletion_request', 'deletion_cancellation', 'reauthentication']` — sem esta correção, todas as novas chamadas eram silenciosamente rejeitadas por `permission-denied`. Review fix: `logDeletionCancellation` agora inclui `'metadata': {}` (campo obrigatório pelo schema `keys().hasOnly()` das rules).

- **Export completo: cápsulas recebidas + coordenadas GPS (`complete_export_service.dart`):** corrigido gap de portabilidade (GDPR Art. 20): o export só incluía cápsulas enviadas (`senderUid == uid`), faltavam as recebidas/participadas. Adicionado `_fetchCapsules()` com query `participantUids array-contains uid` + deduplicação por `byId` map, espelhando o padrão de `_fetchLetters()`. Fotos de cápsulas recebidas agora também são incluídas no ZIP. Corrigido P2 da review: coordenadas GPS agora exportadas como `{latitude, longitude}` em vez de apenas `hasLocation: true` — o utilizador forneceu os dados, portabilidade exige incluí-los. Adicionado helper `_geoPointToMap()` com suporte a `GeoPoint` e Map raw.

- **Re-autenticação social para eliminação de conta (`account_deletion_service.dart`, `settings_screen.dart`):** utilizadores que fizeram login via Apple ou Google não conseguiam eliminar a conta porque `reauthenticateWithPassword()` falhava — não têm password. Adicionado: `currentAuthProvider()` que deteta o provider (prioriza social sobre password para utilizadores com múltiplos providers), `_reauthenticateWithGoogle()` via `GoogleSignIn().signIn()` + `GoogleAuthProvider.credential`, `_reauthenticateWithApple()` via `SignInWithApple` com nonce SHA-256 + `OAuthProvider('apple.com').credential`. O `_DeleteAccountSheet` agora deteta o provider e mostra: campo de password (email/password) OU botão "Verify with Apple/Google" (social). A re-auth social acontece dentro do sheet com feedback visual (spinner, estado "verified", erro). 4 chaves i18n novas em 4 idiomas: `settingsDeleteReauthLabel`, `settingsDeleteReauthApple`, `settingsDeleteReauthGoogle`, `settingsDeleteReauthSuccess`. Review: corrigido missing `await` no `onConfirm`, hardcoded colors dark mode, e provider iteration order.

- **Anonimização automática de feedback — 1 ano (`functions/src/anonymize_old_feedback.ts`):** Cloud Function scheduled (`onSchedule`, diariamente às 05:00 UTC) que cumpre o compromisso da secção 9 da Política de Privacidade: "Product feedback: anonymized after 1 year." Utiliza query de janela temporal (24h): feedback com `createdAt` entre 366 e 365 dias atrás — custo constante independente do volume acumulado. Backfill com `select("anonymizedAt")` + `limit(450)` para feedback anterior ao deploy. Campos PII removidos via `FieldValue.delete()`: `uid`, `message`. Campos estatísticos mantidos: `type`, `createdAt`, `appLocale`, `platform`. Campo `anonymizedAt` adicionado como timestamp de auditoria. Batch writes com limite de 450 operações. Timeout: 300s, retry: 1. Exportada no `index.ts`.

- **Purge automático de logs de moderação — 2 anos (`functions/src/purge_old_moderation_logs.ts`):** Cloud Function scheduled (`onSchedule`, diariamente às 04:30 UTC) que cumpre o compromisso da secção 9 da Política de Privacidade: "Moderation logs: 2 years." Elimina completamente (não anonimiza) documentos de `moderationIncidents` e `moderationReviews` com `createdAt` anterior a 730 dias. Paginação interna: query com `limit(450)` em loop até esgotar documentos elegíveis, evitando OOM em volumes grandes. PII eliminado: `callerUid`, `message` (incidents); `authorUid`, `authorDisplayName`, `text`, `resolvedByUid`, `adminFeedback` (reviews). Timeout: 300s, retry: 1. Exportada no `index.ts`. Sem composite index necessário.

- **Anonimização automática de denúncias — 90 dias (`functions/src/anonymize_resolved_reports.ts`):** Cloud Function scheduled (`onSchedule`, diariamente às 04:00 UTC) que cumpre o compromisso da secção 9 da Política de Privacidade: "Content reports: anonymized 90 days after resolution." Query composta: `status` in [resolved, dismissed] + `resolvedAt` ≤ now − 90 dias + `anonymizedAt` == null. Campos PII removidos via `FieldValue.delete()`: `reporterUid`, `detail`, `resolvedByUid`. Campos estatísticos mantidos: `targetType`, `targetId`, `letterId`, `reason`, `status`, `createdAt`, `resolvedAt`. Campo `anonymizedAt` adicionado como timestamp de auditoria. Batch writes com limite de 450 operações por batch para respeitar o limite Firestore de 500. Timeout: 300s, retry: 1. Exportada no `index.ts`.

- **Exportação completa de dados — GDPR Art. 20 / LGPD Art. 18 (`complete_export_service.dart`, `settings_screen.dart`, `privacy_log_service.dart`):** novo serviço `lib/core/export/complete_export_service.dart` que exporta todos os dados do utilizador num ficheiro ZIP: perfil, cartas (sent + received, deduplicadas), cápsulas (+ fotos), comentários, likes, follows (followers + following), badges — todos em JSON (`whenote-sar-v1` format) + media (voz .m4a, manuscrito .jpg, fotos de cápsulas .jpg). Sanitização de campos internos (UIDs alheios, stripeCustomerId, searchTokens, etc.) com conversão de Timestamps para ISO-8601. URLs de media validadas contra allowlists de Firebase Storage (proteção SSRF). UI no Settings atualizada: restrição Pro removida — export gratuito para todos os utilizadores; novo `_CompleteExportButton` StatefulWidget com `LinearProgressIndicator` e estágios; partilha via `share_plus`. `PrivacyLogService.logCompleteExport()` adicionado para audit trail com metadata detalhada (contagens por tipo). ### Fixed

- **Cursor preso + menu copiar/colar ausente no campo de mensagem da carta (`write_letter_screen.dart`):** dois problemas distintos com a mesma raiz. (1) O `GestureDetector(onTap: setState(_showResults=false))` envolvia o `SingleChildScrollView` inteiro — o gesto de toque capturava o evento antes do `TextField`, impedindo o reposicionamento do cursor ao tocar no meio do texto e suprimindo o menu de contexto nativo (Copiar / Colar) acionado por toque longo. (2) No estado colapsado, o campo de mensagem era um widget `Text` estático (pré-visualização read-only), sem `TextEditingController` ativo — não havia cursor real nem integração com o clipboard do sistema. **Correção:** campo de mensagem unificado num único `TextField` com `minLines: 1` / `maxLines: _messageExpanded ? 8 : 1`, `enableInteractiveSelection: true`; `FocusNode` listener `_onMessageFocusChanged()` regista-se em `initState` e remove-se em `dispose` — ao receber foco com campo colapsado, expande automaticamente e faz `Scrollable.ensureVisible`; `GestureDetector` global removido e substituído por `TapRegion` isolado apenas na zona de busca de destinatário + lista de resultados, com `onTapOutside` adiado via `addPostFrameCallback` para não cancelar gestos longos noutros campos; `isDense` removido (no-op com `contentPadding` explícito); `contentPadding` diferenciado por estado (`fromLTRB(16,0,16,16)` expandido; `symmetric(h:16,v:12)` colapsado). Bloqueador **B2** do MVP fechado.

### Changed

- **Menção ao Global Privacy Control (GPC) removida da Política de Privacidade:** frase "We honor the Global Privacy Control (GPC) signal as a valid opt-out request under CCPA" removida da secção 15 (Tracking Technologies) nos 4 idiomas (en/pt/pt-BR/es). GPC é um header HTTP de browser (`Sec-GPC: 1`) — não aplicável a apps nativos móveis. A promessa era inimplementável. O opt-out de analytics para utilizadores CCPA fica coberto pelo toggle nas Settings. Menção será re-adicionada quando existir versão web com implementação real.

- **SMTP customizado migrado de SendGrid para Google Workspace SMTP Relay (2026-04-26):** o envio de emails de autenticação Firebase (verificação de email, reset de password, email change) foi migrado de SendGrid (`smtp.sendgrid.net`) para o SMTP Relay do Google Workspace (`smtp-relay.gmail.com`). Configuração feita em duas etapas: (1) Google Admin Console → Apps → Google Workspace → Gmail → Routing → SMTP relay service: allowed senders "Any addresses", "Require SMTP Authentication" ✅, "Require TLS encryption" ✅; (2) Firebase Console → Authentication → Templates → SMTP settings: host `smtp-relay.gmail.com`, porta 587, STARTTLS, username `yurilima@whenote.com`, password = App Password gerada na conta Workspace. Sender address alterado para `noreply@whenote.com`. Pré-requisito: 2-Step Verification ativada na conta Workspace. Motivação: eliminar dependência do SendGrid e simplificar stack usando serviço já incluído no Google Workspace.
