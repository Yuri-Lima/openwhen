DETAILED FIX FOR WARNING #1: "Missing entitlement: push-to-talk"
================================================================

ROOT CAUSE:
You have 'push-to-talk' listed in your UIBackgroundModes array, but your app does not have the corresponding 'com.apple.developer.push-to-talk' entitlement from Apple. This is a capability mismatch.

SOLUTION:
Open ios/Runner/Info.plist and find the UIBackgroundModes key. It looks like:

  <key>UIBackgroundModes</key>
  <array>
    <string>push-to-talk</string>
    <!-- other modes may be here -->
  </array>

DELETE the entire line:
  <string>push-to-talk</string>

Since your app does not implement push-to-talk functionality, this background mode should not be declared.

---

DETAILED FIX FOR WARNING #2: "Missing purpose string NSLocationAlwaysAndWhenInUseUsageDescription"
==================================================================================================

ROOT CAUSE:
You have 'location' in your UIBackgroundModes array. When location is declared as a background mode, iOS requires BOTH of these keys in Info.plist:
  - NSLocationWhenInUseUsageDescription (you already have this)
  - NSLocationAlwaysAndWhenInUseUsageDescription (MISSING)

Even though your app only requests "when in use" permission at runtime, Apple's validation requires the "always" description to be present when location is a background mode. This is documented in Phase 3b of the flutter-apple-publish skill.

SOLUTION:
Add this key to ios/Runner/Info.plist if it is not already present:

  <key>NSLocationAlwaysAndWhenInUseUsageDescription</key>
  <string>Your app uses your location in the background to provide [describe the feature, e.g., 'location-based notifications', 'tracking', etc.]</string>

Example values you could use:
  - "Your location is used to provide location-based services even when the app is not in use."
  - "We access your location in the background to [specific feature name]."
  - "Background location access is needed for [your specific use case]."

The string must be non-empty. Choose a description that accurately reflects how your app uses background location.
