FLUTTER IPA UPLOAD ERROR FIX: ITMS-90396 Missing BGTaskSchedulerPermittedIdentifiers
================================================================================

PROBLEM SUMMARY
---------------
Error: "ERROR ITMS-90396: Missing BGTaskSchedulerPermittedIdentifiers. The Info.plist
in the Runner.app bundle contains the processing value for the UIBackgroundModes key,
but no BGTaskSchedulerPermittedIdentifiers were found."

ROOT CAUSE
----------
Your iOS app declares UIBackgroundModes in Info.plist (for location, audio, remote
notifications, Bluetooth, VoIP, etc.) but does NOT include a BGTaskSchedulerPermittedIdentifiers
entry. Apple's Transporter validation requires that if UIBackgroundModes is present,
BGTaskSchedulerPermittedIdentifiers must also be defined (even if empty).

The UIBackgroundModes is likely declared by your dependencies:
- firebase_messaging (remote notifications, background processing)
- geolocator (location background mode)
- just_audio / audio_session (audio background mode)

SOLUTION
--------
Add BGTaskSchedulerPermittedIdentifiers to ios/Runner/Info.plist. This can be:
1. Empty array (if you don't actually use BGTaskScheduler)
2. Array with specific identifiers (if you do use them)

STEP-BY-STEP FIX
----------------

1. Open ios/Runner/Info.plist in your editor

2. Find the UIBackgroundModes section (should be around line 75-86)

3. After the closing </array> tag for UIBackgroundModes, add this:

   <key>BGTaskSchedulerPermittedIdentifiers</key>
   <array/>

   OR if you do use background task scheduling, add specific identifiers:

   <key>BGTaskSchedulerPermittedIdentifiers</key>
   <array>
       <string>com.example.app.backgroundtask</string>
       <!-- add more as needed -->
   </array>

4. Your Info.plist should look like this (showing relevant section):

   <key>UIBackgroundModes</key>
   <array>
       <string>remote-notification</string>
       <string>bluetooth-peripheral</string>
       <string>audio</string>
       <string>fetch</string>
       <string>external-accessory</string>
       <string>location</string>
       <string>bluetooth-central</string>
       <string>nearby-interaction</string>
       <string>voip</string>
   </array>
   <key>BGTaskSchedulerPermittedIdentifiers</key>
   <array/>

5. Save the file

6. Rebuild your IPA:
   flutter clean
   flutter pub get
   flutter build ipa --release

7. Re-upload to Transporter - error should be resolved

IMPORTANT NOTES
---------------
- The empty <array/> is sufficient unless your app explicitly uses BGTaskScheduler
- If your app uses background task scheduling in native Swift/Kotlin code, add the
  specific identifiers you defined in your native code
- The identifier format is typically: com.yourcompany.app.taskName
- Do NOT remove UIBackgroundModes unless you stop using those background capabilities
- This is an Apple requirement starting from iOS 13+ for proper background mode validation

WHY THIS HAPPENS
-----------------
Apple enforces a strict requirement: any app declaring background modes via UIBackgroundModes
must explicitly list which Background Task Scheduler identifiers it uses. This prevents apps
from silently consuming resources in the background without proper disclosure.

If you don't use BGTaskScheduler at all (which is the case for most apps that just need
location, audio, or push notifications), declare an empty array to comply with this requirement.
